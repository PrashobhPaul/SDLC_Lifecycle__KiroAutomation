
## File-split rules per module
- `main.tf` — resources only.
- `variables.tf` — every input variable with type + description + validation.
- `outputs.tf` — every consumable output, with description.
- `versions.tf` — `required_version`, `required_providers`.
- `README.md` — auto-generated by `terraform-docs`; never hand-edited.

## Naming
- Resource names: `<project>_<bounded_context>_<purpose>` — `calm_member_pilot_table`.
- Variables: `snake_case`.
- Tags: same casing as the AWS account standard (typically `Project`, `Environment`).

## Validation
Every variable has explicit `type` and at least one `validation` block where the value space is bounded:
```hcl
variable "lambda_memory_mb" {
  type        = number
  description = "Lambda memory in MB"
  default     = 512
  validation {
    condition     = var.lambda_memory_mb >= 128 && var.lambda_memory_mb <= 10240
    error_message = "Lambda memory must be 128–10240 MB."
  }
}
```

## Module outputs
- Outputs expose only what downstream consumers need (BE rule, MAJOR if over-exposed).
- ARNs always; ids only when needed.
- Sensitive outputs marked `sensitive = true`.
