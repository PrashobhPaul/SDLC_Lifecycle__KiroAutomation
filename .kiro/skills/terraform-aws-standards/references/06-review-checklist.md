# Terraform Review Checklist

## Tooling pass (Code Reviewer runs in parallel)

```bash
tfsec --soft-fail-warning terraform/                      # security
tflint --recursive --format json terraform/               # syntax + best practice
checkov -d terraform/ --framework terraform               # compliance
terraform fmt -check -recursive terraform/                # formatting
terraform validate                                        # syntax
terraform plan -input=false -out=plan.bin terraform/      # plan
node tools/iam-sdk-crossref.ts ...                        # IAM↔SDK cross-reference
```

## Per-file checklist (BE audit Section I + meeting-notes context)

### Module structure
- [ ] `main.tf`, `variables.tf`, `outputs.tf`, `versions.tf` present
- [ ] `README.md` is generated by `terraform-docs`, not hand-edited
- [ ] No region/account/ARN hardcoded (BE rule 13 — CRITICAL)
- [ ] Consistent tags applied: `Project`, `Environment`, `Owner`, `BoundedContext`, `CostCenter`

### Lambda
- [ ] runtime = `nodejs22.x` (or current LTS); deprecated runtimes flagged (BE rule 62)
- [ ] architecture = `arm64` unless documented (BE rule 63)
- [ ] memory right-sized; not 128MB for DB/HTTP
- [ ] timeout aligned with downstream
- [ ] reserved_concurrent_executions where required
- [ ] DLQ configured (BE rule 16)
- [ ] CloudWatch log group with retention (BE rule 46)
- [ ] X-Ray tracing on (BE rule 17)
- [ ] `NODE_OPTIONS=--enable-source-maps` (BE rule 65)
- [ ] `AWS_NODEJS_CONNECTION_REUSE_ENABLED=1`
- [ ] Bundle size unzipped < 250 MB / zipped < 50 MB (BE rule 64)

### IAM
- [ ] No `Action: "*"` AND `Resource: "*"` together (CRITICAL)
- [ ] Every IAM action ↔ corresponding SDK call (BE rule 61)
- [ ] Permission boundaries on prod roles
- [ ] No inline policies in prod
- [ ] Cross-account trust uses ExternalId where applicable

### DynamoDB
- [ ] Global Tables on data the Calm UI relies on (multi-region requirement)
- [ ] Encryption at rest with customer-managed KMS key
- [ ] PITR enabled in prod
- [ ] No `BillingMode = "PROVISIONED"` without an explicit capacity plan

### Aurora
- [ ] Aurora Global Database for the Calm cluster
- [ ] RDS Proxy in front for Lambda connections
- [ ] Encryption at rest
- [ ] Automated backups + cross-region snapshot copy

### AppSync
- [ ] Lambda authorizer configured (per meeting notes)
- [ ] Logging enabled (FieldExecution / FieldRequestResponse where appropriate)
- [ ] X-Ray tracing on
- [ ] No throttling at AppSync (per meeting notes); scaling happens at Lambda

### Kafka
- [ ] Topics replicated cross-region via MM2 (multi-region requirement)
- [ ] Consumer groups documented per topic
- [ ] DLQ topic for unprocessable messages
- [ ] Schema Registry connection if Avro/Protobuf

### S3
- [ ] Bucket encryption: SSE-KMS with CMK
- [ ] Versioning + MFA-delete in prod
- [ ] Cross-region replication on critical buckets
- [ ] No public ACL; bucket policy restricts to known principals

### CloudWatch
- [ ] Alarms for error rate, throttles, p99 duration, DLQ depth (BE checklist)
- [ ] Log groups with retention
- [ ] EMF used for custom metrics where appropriate

### CI gates
- [ ] tfsec runs in CI (BE checklist)
- [ ] tflint runs in CI
- [ ] terraform plan runs in CI for drift detection
- [ ] Plan output attached to G4 evidence

## Severity mapping

| Finding | Severity |
|---------|----------|
| `Action: "*"` AND `Resource: "*"` | CRITICAL |
| Hardcoded region/account/ARN | MAJOR (CRITICAL if it leaks credentials) |
| Missing KMS encryption on DynamoDB/S3/SQS/SNS | CRITICAL |
| Open SG (`0.0.0.0/0`) on prod | CRITICAL |
| Missing CloudWatch retention | MAJOR (BE rule 46) |
| Missing DLQ on async trigger | MAJOR (BE rule 16) |
| Deprecated Lambda runtime | MAJOR (BE rule 62) |
| Bundle > 50MB zipped | MAJOR (BE rule 64) |
| No `arm64` justification for `x86_64` | MAJOR (BE rule 63) |
| State not separated per env | MAJOR (BE rule 66) |
| `tfsec --soft-fail-warning` warnings | MAJOR per rule |
